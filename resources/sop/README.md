# SOP 触发指令卡片总入口

## 这是什么

针对 AI 在新会话中「不按 StudyFlow 执行」的问题，把每个触发指令拆成**强制可勾选的 SOP 卡片**。

**与 StudyFlow.md 的关系**：
- StudyFlow.md = 规则全集（解释为什么这么做）
- SOP/*.md = **执行清单**（每步必须做、禁止做、模板原文）

**冲突时**：SOP 卡片优先，按硬词执行。

## 强制读取规则（任何触发指令生效前）

AI 收到任何触发指令后，**第一件事**是按顺序 Read 以下文件：

1. `docx/SOP/_GLOBAL_RULES.md`（全局规则，含「继续」歧义消除）
2. `docx/SOP/SOP_<指令名>.md`（对应卡片）

未读取上述文件就开始执行任何步骤 = **违规**，必须立即停止并补读。

## 触发指令 → SOP 卡片映射

| 用户输入 | SOP 卡片 |
|---------|---------|
| `[开始今日学习]` | `SOP_开始今日学习.md` |
| `[恢复学习]` | `SOP_恢复学习.md` |
| `[下一内容]` / "继续"（仅在歧义场景） | `SOP_下一内容.md` |
| `[超前学习]` | `SOP_下一内容.md` |
| `[同步] XXX` | `SOP_同步.md` |
| `[开始写代码]` | `SOP_开始写代码.md` |
| `[开始今日复盘]` | `SOP_开始今日复盘.md` |
| `[结束今日学习]` | `SOP_结束今日学习.md` |

## SOP 卡片通用结构

每张卡固定 6 块：

1. **触发场景** — 用户说什么时启用
2. **执行前必读（MUST READ）** — 必须 Read 的文件清单
3. **FAIL-FAST 自检** — 不满足条件就 STOP
4. **步骤（MUST 按顺序）** — 不允许跳步、不允许并行
5. **必输模板（原文照搬）** — 复制即用，不允许改格式
6. **禁止行为（FORBIDDEN）** — 列出常见违规
7. **完成判据（POST-CONDITIONS）** — 自检通过才算完

## 硬词使用规则

| 用词 | 含义 |
|------|------|
| **必须 / MUST** | 不做就是违规 |
| **禁止 / MUST NOT / FORBIDDEN** | 做了就是违规 |
| **STOP** | 立即停止，等用户回应 |
| **FAIL** | 自检不通过，整个流程退出 |
| **照搬 / 原文** | 一字不改，不允许"等价改写" |

`StudyFlow.md` 中所有「应该 / 建议 / 推荐」在 SOP 卡片中已全部硬化为「必须 / 禁止」。**SOP 与 StudyFlow 不一致时，以 SOP 为准。**

## 全局规则速查（详见 _GLOBAL_RULES.md）

| # | 规则 | 一句话 |
|---|------|-------|
| 1 | 「继续」歧义消除 | "继续" ≠ `[下一内容]`，模糊时反问 |
| 2 | 掌握情况检查必输 | 每个单元结束必输 5 字段固定块 |
| 3 | StudyFlow.md 不是参考 | 所有"应该"读作"必须" |
| 4 | 自动保存中间状态 | 单元/同步/确认后立即写 StudyMemory |
| 5 | FAIL-FAST 前置门禁 | 自检不过 STOP，禁止绕过 |
| 6 | 触发指令必须读 SOP | 新会话第一个工具调用必须是 Read SOP |
| 7 | **Markdown 标记 = 必输** | `##` `**` `[ ]` `---` `\|` `>` 模板里有就必须输出 |
| 8 | **SOP > StudyFlow** | 模板冲突时永远以 SOP 为准，禁止以 StudyFlow 为依据省略 SOP 字段/字符 |
| 9 | **用户面输出 ≡ 文件 Edit** | 直接给用户的 assistant message 与 Edit 写文件 Markdown 一视同仁，禁止"美化为纯文本"；禁止自检欺骗 |
| 10 | **模板字面锁定（Template Lock）** | SOP 模板 = 不可变字符串，仅 `<占位符>` 可替换，禁止"语义等价改写" |
| 11 | **深度讲解黄金法则** | 核心技术点必须含 Why（架构权衡）/ Where（源码定位）/ QA（面试三连问） |
| 12 | **Day 21 代码行数特赦** | 仅 Day 21 豁免单次 ≤20 行代码限制 |
| 13 | **掌握度分级量化** | 2 回合连环追问 + 1.0-5.0 量化评分，默认"需巩固"，禁止自嗨式判断 |
| 14 | **状态变更验证 Hook** | 备份 → 写入 → 运行 validate_study.py → 失败回滚 |
