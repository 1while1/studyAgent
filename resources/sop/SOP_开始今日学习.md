# SOP: [开始今日学习]

## 触发场景

用户说 `[开始今日学习]`，开启新一天的学习流程。

---

## 执行前必读（MUST READ）

按顺序 Read（**禁止**跳过）：
1. `docx/StudyState.json`（核心学习状态文件，以提取 current_day 与进度）
2. `docx/SOP/_GLOBAL_RULES.md`
3. `docx/SOP/SOP_开始今日学习.md`（本卡）
4. `docx/StudyFlow.md` 第 1 节（节奏）+ 第 2 节（4 步流程）
5. `docx/Study.md`（如不存在按 Step 1 初始化）
6. 最近 1 天的 `docx/StudyMemory/` 文件（即当前 Day N 对应的 Day_N.md，不存在则跳过）
7. `docx/DocIndex.md`（获取文档实际路径）

**禁止**：未 Read 上述 7 个文件就执行任何后续步骤。

### 「重新开始今日学习」分支的数据保留规则（关键）

用户选择"重新开始今日学习"时，Edit StudyMemory 必须遵守：

| 字段 | 重置规则 |
|------|---------|
| 今日导学单元 `[x]` / `[ ]` | **允许**全部重置为 `[ ]` 或保留已完成状态由用户决定 |
| 单元行的「备注 / 已掌握细节」 | **禁止**覆盖或简化（如"已掌握，用户自述准确，概念2有小偏差已纠正" → 不能改成"前次已掌握"） |
| [同步] 记录 4 行（已掌握/卡壳/疑问/代码完成） | **禁止**清空，必须原样保留 |
| 掌握度评分 | **禁止**重置 |
| ragent-replica 进度 | **禁止**重置 |
| AI 拷打评语 | **禁止**重置 |
| 推荐论文阅读情况 | **禁止**重置 |
| 明日优先项 | **允许**根据新计划更新 |

**违规判定**：Edit diff 中如果出现"前次详细笔记被改短"或"[同步] 记录被清空"= 违规，必须立即回滚 Edit 并重新写入。

## 步骤（MUST 按顺序，4 步流程）

### Step 1: 分析历史文档

**必做**：
1. Read `docx/StudyState.json`
   - 不存在 → 初始化创建（`current_day=1`，整体完成度 0%，`active_day_completed=false`）
   - 存在 → 确认 `current_day`，并校验其 `active_day_completed` 是否为 true。如果为 false，提示“当前活跃天 Day <current_day> 尚未完成，您是需要 [恢复学习] 还是 [重新开始今日学习]？”。
2. Read `docx/Study.md`
   - 存在 → 确认 "当前是 Day N"
3. Read 最近 3 天的 StudyMemory (以 JSON 映射的 `Day_0X.md` 文件为准)
3. 提取昨日"明日优先项"和"评分 < 3 的回滚项"

**必输模板**：

<!-- template:step1_history -->
```
---【Step 1：历史分析】---
当前进度：Day <N> / 25
昨日学习：<日期> 完成单元数 <X>
昨日卡壳：<列表>
昨日 < 3 分回滚项：<列表>

将优先安排：<列表>
---
```
<!-- /template:step1_history -->

### Step 2: 扫描仓库校验 Project.md

**Day 1 特殊处理**：
- 全量扫描 `ragent原项目/` 目录结构和核心包
- 用 Glob 获取主要模块路径
- 生成 / 重写完整 Project.md

**Day ≥ 2 处理**：
- 用 Glob 校验今日涉及模块的核心类是否存在
- 与 Project.md 中描述对比，有差异则 Edit 修正
- 无差异跳过

**必输模板**：

<!-- template:step2_repo_check -->
```
---【Step 2：仓库校验】---
今日涉及模块：<列表>
扫描结果：[✅ 一致 / ⚠️ 已修正 X 处 / ❌ 严重不符]
（如有差异）修正内容：<列表>
---
```
<!-- /template:step2_repo_check -->

### Step 3: 生成今日计划

**必做**：
1. Read `docx/Study.md` 中 Day N 的内容大纲
2. Read `docx/DocIndex.md` 获取今日文档实际路径
3. 把大纲细化为 3-5 个学习单元
4. Edit `Study.md`，在 Day N 下追加详细单元列表（格式见 StudyFlow.md 第 13 节）
5. 初始化 `docx/StudyState.json` 中的当前 Day 状态，并追加单元列表。
6. 创建当日 StudyMemory 文件，路径为 `docx/StudyMemory/Day_<NN>.md` (例如 `Day_01.md`)，并填入 StudyFlow.md 第 11 节的模板。

**必输模板**（原文照搬，**Markdown 标记字符全部必输**）：

<!-- template:step3_plan -->
```
---【Step 3：今日计划】---
## Day <N> | <YYYY-MM-DD>（星期X）
**目标**：<1 句话>

**导学单元**：
1. [ ] 单元A：<标题>（预计 <X>min）
   - 文档：<路径>
   - 核心知识点：<3-5 个>
2. [ ] 单元B：<标题>（预计 <X>min）
   - 文档：<路径>
   - 核心知识点：<3-5 个>
3. [ ] 单元C：<标题>（预计 <X>min>
   ...

**编码目标**：ragent-replica 完成 <模块>
**推荐论文**：《<论文名>》— 重点读 Section <X>
**面试话术目标**：产出"<话题>"的 30 秒/2 分钟版回答

总预计时长：<5-6h>
---
```
<!-- /template:step3_plan -->

**输出前自检清单**（任一不通过禁止输出，按 `_GLOBAL_RULES.md` 规则 7）：
- [ ] `## Day <N>` 二级标题保留（**不允许**改成纯文本 `Day <N>`）
- [ ] `**目标**：` `**导学单元**：` `**编码目标**：` `**推荐论文**：` `**面试话术目标**：` 5 处加粗全部保留
- [ ] 每个单元行开头的 `[ ]` 复选框保留（不允许省略）
- [ ] 每个单元下方的 `- 文档：` 和 `- 核心知识点：` 两行子项保留
- [ ] 末尾的 `---` 闭合保留
- [ ] 占位符 `<N>` `<YYYY-MM-DD>` `<标题>` 等已替换为实际值

> 📁 违规示例、完整 Worked Example 与逐字符比对要点见 `docx/SOP/ViolationHistory.md` §3。

**当 SOP 模板与 StudyFlow.md 第 13 节冲突时**：按 SOP（规则 8）。**禁止**以"StudyFlow.md 第 13 节没有 `##` / `**` / `[ ]`"为理由省略这些字符。

### Step 4: 开始 AI 导学

**Step 4 输出顺序（3 个子块，按顺序输出，每块独立闭合）**：
1. **子块 A**：Step 4 引导头（短）
2. **子块 B**：推荐论文/文章模板（必输，1-2 篇）
3. **子块 C**：单元开场模板（必输，仅 1 段开讲）

**禁止合并 3 个子块**。每个子块必须有独立的 `---【...】---` 开头和 `---` 末尾。

#### 子块 A：Step 4 引导头（原文照搬）

<!-- template:step4_guide -->
```
---【Step 4：开始 AI 导学】---
我先推荐今日论文/文章（子块 B），然后开启第一个学习单元（子块 C）。
---
```
<!-- /template:step4_guide -->

**字面禁止变体（按规则 10 模板锁定原则）**：4 种实测违规变体见 `docx/SOP/ViolationHistory.md` §4，禁止重犯。

**唯一合规输出**（4 行字符必须 1:1 一致）：

```
---【Step 4：开始 AI 导学】---
我先推荐今日论文/文章（子块 B），然后开启第一个学习单元（子块 C）。
---
```

#### 子块 B：推荐论文/文章必输模板（原文照搬）

<!-- template:paper_block -->
```
---【今日推荐论文/文章】---
1. **《<论文/文章名>》**（<作者>, <年份>）
   - 重点章节：Section <X>（<章节主题>）
   - 面试价值：<1-2 句话，说明读完能答什么面试题>
2. **《<论文/文章名>》**（<作者>, <年份>）
   - 重点章节：Section <X>（<章节主题>）
   - 面试价值：<1-2 句话>
---
```
<!-- /template:paper_block -->

**硬约束**：
- 至少 1 篇，最多 2 篇
- 每篇必含 3 个子项：`**《》**` 加粗书名 + `- 重点章节：` + `- 面试价值：`
- Day 1 也必须输出，**禁止**以 "Day 1 是基础概念" 为由跳过（按规则 3，"推荐"读作"必须"）
- **必须**作为独立子块输出，开头 `---【今日推荐论文/文章】---`，末尾 `---`。**禁止**与子块 A 合并

> 📁 Worked Example 与实测违规对照见 `docx/SOP/ViolationHistory.md` §5。



#### 子块 C：单元开场必输模板（**必须**原文照搬，含开头和末尾的 `---`，含所有 `**` 加粗）

<!-- template:unit_open -->
```
---【单元 1 开始】---
单元：<标题>
目标：<1 句话>
预计时长：<X>min
关联面试考点：<1-2 个>

我将带读以下材料：
- 项目文档：<路径>
- 业内论文/文章：<引用>

第一段：<开始讲解，1 段，≤ 300 字>
---
```
<!-- /template:unit_open -->

**模板硬约束**（违反任一 = 违规，按 Rule 10）：
- 开头必须有 `---【单元 N 开始】---`
- **末尾必须有 `---`**（与开头闭合，标记单元开场块结束）— 实测 2026-05-18 两轮都缺末尾 `---`
- 末尾 `---` 之后才能开始正式讲解内容
- 缺末尾 `---` = 单元开场块未闭合 = **违规**
- **「第一段」字段内容硬上限：1 个自然段（≤ 300 字）**。需要讲多段时，**必须**先输出末尾 `---` 闭合单元开场块，再以新段落继续讲解
- **禁止**把「面试怎么问/这里有个关键认知/好，第一个概念讲完...」等后续讲解段塞进末尾 `---` 之前（2026-05-18 实测连续两轮违规：5 段讲解全在末尾 `---` 之前，且末尾 `---` 完全缺失）

**字符级输出顺序强制（按 Rule 10）**：

```
顺序 1: ---【单元 1 开始】---
顺序 2: 单元：<标题>
顺序 3: 目标：<1 句话>
顺序 4: 预计时长：<X>min
顺序 5: 关联面试考点：<1-2 个>
顺序 6: 空行
顺序 7: 我将带读以下材料：
顺序 8: - 项目文档：<路径>
顺序 9: - 业内论文/文章：<引用>
顺序 10: 空行
顺序 11: 第一段：<≤ 300 字自然段>
顺序 12: ---                  ← 关键！必须立即输出这一行
顺序 13: 空行
顺序 14: <第二段讲解>          ← 必须在 --- 之后
顺序 15: <第三段讲解>
...
```

**生成规则**：输出完顺序 11 的「第一段」后，**下一个 token 必须是换行 + `---` + 换行**。如果 AI 想继续讲解但还没输出顺序 12 的 `---`，**必须先输出 `---`，再开始新段落**。



> 📁 违规示例（未闭合 / 第一段溢出）见 `docx/SOP/ViolationHistory.md` §6。

**禁止**：跳过推荐论文环节直接开始讲单元内容。
