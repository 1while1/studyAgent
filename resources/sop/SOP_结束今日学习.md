# SOP: [结束今日学习]

## 触发场景

用户说 `[结束今日学习]`，收尾当日学习并产出复盘资料。

---

## 执行前必读（MUST READ）

按顺序 Read：
1. `docx/StudyState.json`（核心学习状态文件）
2. `docx/SOP/_GLOBAL_RULES.md`
3. `docx/SOP/SOP_结束今日学习.md`（本卡）
4. `docx/StudyFlow.md` 第 9 节（结束流程）+ 第 11 节（StudyMemory 模板）+ 第 12 节（InterviewQA 模板）
5. 对应活跃天的 `docx/StudyMemory/Day_N.md`
6. `docx/Study.md`
7. `docx/InterviewQA.md`

---

## FAIL-FAST 自检

- [ ] `docx/StudyState.json` 是否存在，且当前活跃天对应的 `docx/StudyMemory/Day_N.md` 是否存在？（不存在则提示"今日还没 `[开始今日学习]`，无内容可结束"）
- [ ] 当日是否已做 `[开始今日复盘]`？
  - 否 → **强制提醒**："今日尚未复盘，建议先 `[开始今日复盘]` 再结束。要跳过复盘直接结束吗？回 [跳过复盘] / [先复盘]"
  - 是 → 继续
- [ ] StudyMemory 中是否至少有 1 个 `[x]` 已完成单元？（无则提示"今天还没完成任何单元，确定结束？"）

---

## 步骤（MUST 按顺序，6 步全做）

### Step 1: 汇总当日 `[同步]` 记录

Read 当日 StudyMemory，提取：
- 已掌握列表
- 卡壳列表
- 疑问列表（已解答 / 待解答）
- 代码完成列表

**必输模板**：

<!-- template:end_step1_sync -->
```
---【Step 1：[同步] 汇总】---
今日 [同步] 统计：
- 已掌握：<N> 项
- 卡壳：<N> 项（其中已解决 <X> 项）
- 疑问：<N> 项（已解答 <X> / 待解答 <Y>）
- 代码完成：<N> 模块

(如有「待解答」)⚠️ 仍有 <Y> 个待解答疑问，建议补充 `[同步] 疑问 XXX` 解答完再结束。
---
```
<!-- /template:end_step1_sync -->

**输出前自检清单**（任一不通过禁止输出，按 `_GLOBAL_RULES.md` 规则 7）：
- [ ] 开头 `---【Step 1：[同步] 汇总】---` 保留（含 `[同步]` 方括号）
- [ ] 4 个 `- ` 列表项保留：`- 已掌握：` `- 卡壳：` `- 疑问：` `- 代码完成：`
- [ ] `⚠️` emoji 警告保留（如适用）
- [ ] 行内代码 `` `[同步] 疑问 XXX` `` 反引号保留
- [ ] 末尾 `---` 闭合保留

### Step 2: 完善 StudyMemory 与 JSON 状态（必按模板）

1. 更新 `docx/StudyState.json` 中的各单元完成情况，如有未完单元，将状态设为 `"postponed"`。标记 `active_day_completed = true`。
2. Edit 当日 `docx/StudyMemory/Day_N.md`，确保所有字段齐全（StudyFlow.md 第 11 节模板的 7 个字段）：

- [ ] 今日导学单元（每个单元 `[x]` / `[ ]` 标记）
- [ ] [同步] 记录（4 行：已掌握 / 卡壳 / 疑问 / 代码完成）
- [ ] 掌握度评分（每个知识点 1-5 分）
- [ ] ragent-replica 进度（已完成 / 今日新增 / 待完成）
- [ ] AI 拷打评语（强项 / 风险点 / 建议）
- [ ] 推荐论文/文章阅读情况（标记已读章节）
- [ ] 明日优先项（含 < 3 分回滚项）

**禁止**：任一字段为空（无内容写"无"，不允许整段缺失）。

### Step 3: 追加面试话术到 InterviewQA.md

Read 当日对话，识别用户产出的优秀面试话术（来自 `[同步] 面试话术` 或拷打中精彩回答）。

对每条话术：
1. 按 StudyFlow.md 第 12 节模板追加到 `InterviewQA.md`
2. 追问预案 ≥ 3 个（不足则 AI 补足）
3. 标注来源日期和场景

**必输模板**：

<!-- template:end_step3_qa -->
```
---【Step 3：面试话术追加】---
今日追加到 InterviewQA.md：
1. <问题标题 1>
2. <问题标题 2>
...

总计 <N> 条面试话术。
---
```
<!-- /template:end_step3_qa -->

**输出前自检清单**（任一不通过禁止输出，按 `_GLOBAL_RULES.md` 规则 7）：
- [ ] 开头 `---【Step 3：面试话术追加】---` 保留
- [ ] 问题列表 `1. 2.` 数字编号保留
- [ ] 末尾 `---` 闭合保留

如果今日无新话术，输出"今日未产生新面试话术，建议明日有意识地输出 30 秒/2 分钟版回答"。

### Step 4: 生成 StudyReview/Day_N-<模块名>.md

**必做**：创建详细复盘资料，**字数 ≥ 3000 字**。

文件路径：`docx/StudyReview/Day_<NN>-<主模块名>.md` (例如 `Day_01-大模型核心概念.md`)

**必含 7 个章节**（缺一不可）：

<!-- template:study_review_doc -->
```markdown
# <模块名> 详细复盘 - <YYYY-MM-DD>

## 1. 今日涉及的所有代码文件
- 文件路径 1：<完整路径>
  - 核心代码片段：（贴出 10-30 行关键代码）
  - 作用：<2-3 句话>
- 文件路径 2：...

## 2. 业务/技术流程图
（用文字描述 + ASCII 图，或描述待补图的位置）
- 调用链路：A → B → C
- 状态流转：S1 → S2 → S3
- 关键决策点：...

## 3. 关键代码逐行详解
逐行讲解关键方法，格式：
- `<文件>:<行号>` <代码>
  - 作用：...
  - 设计意图：...
  - 面试考点：...

## 4. 面试话术
### 精简版（30 秒）
<内容>

### 展开版（2 分钟）
<内容>

### 追问预案（≥ 3 个）
- Q1: ...
  A1: ...
- Q2: ...
  A2: ...
- Q3: ...
  A3: ...

## 5. 推荐的论文/文章
- 《<名>》第 X 章
  - 核心观点：...
  - 与项目关联：...

## 6. ragent-replica 编码进度
- 已完成模块：...
- 今日新增代码：<文件 + 行数>
- 待完成项：...

## 7. 卡壳点记录和明日复习重点
- 卡壳 1：...（明日如何巩固）
- 卡壳 2：...
- 明日 priority 列表：...
```
<!-- /template:study_review_doc -->

**写入前自检清单**（任一不通过禁止落盘，按 `_GLOBAL_RULES.md` 规则 7）：
- [ ] 一级标题 `# <模块名> 详细复盘 - <YYYY-MM-DD>` 保留 `#`（不可改为纯文本）
- [ ] **7 个二级标题** `## 1.` ~ `## 7.` 全部保留 `##` + 编号（缺一即违规）
- [ ] Step 4 子章节 `### 精简版（30 秒）` `### 展开版（2 分钟）` `### 追问预案（≥ 3 个）` 保留 `###`
- [ ] 代码片段用三反引号 ` ``` ` 包裹，不可去掉
- [ ] 列表项 `- ` 保留（不可改为纯文本罗列）
- [ ] 行内代码 `` `<文件>:<行号>` `` 反引号保留
- [ ] 占位符 `<模块名>` `<YYYY-MM-DD>` `<完整路径>` 等已替换为实际值

**字数检查**：写完后用 wc 或目测确认 ≥ 3000 字，不足则补充章节深度。

### Step 5: 更新 Study.md 完成状态与顺延计划

Edit `docx/Study.md`：
- Day N 标题加上 ✅ 完成标记。
- 整体完成度百分比 +4%（每日 1/25）。
- **顺延处理**：如有顺延单元，在其大纲首行自动追加：`- [ ] 顺延单元：<顺延单元名称>`。
- 并在 `docx/StudyState.json` 中，将这些顺延单元初始化写入 Day N+1 的待学列表中。

- ❌ Step 4 章节缺失（必须 7 个全在）
- ❌ Step 4 追问预案 < 3 个
- ❌ 跳过 Step 5 不更新 Study.md
- ❌ 跳过 Step 6 不输出明日预告
- ❌ 用户尚未复盘且选了 `[先复盘]` 时强行结束
