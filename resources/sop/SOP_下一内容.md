## 步骤（MUST 按顺序，禁止合并）

### Step 1: 输出「掌握情况检查」与第一轮检验（第 1 回合）

**进入此 SOP 后，AI 在当前回复中必须首先输出检查块（模板见 `_GLOBAL_RULES.md` 规则 2，禁止变形）**：

<!-- template:mastery_check -->
```
---【掌握情况检查】---
当前学习单元：<填入>
- 已讲解知识点：<填入>
- 用户提问/卡点：<填入>
- 掌握度评估：[已掌握 / 基本掌握 / 需巩固]
- 编码进度：[已完成 / 进行中 / 未开始 / 不适用]

请确认：
1. 继续下一内容（说 [下一内容]）
2. 再巩固一下当前内容
3. 我有新问题要问
4. 开始写代码（说 [开始写代码]）
---
```
<!-- /template:mastery_check -->

**【第 1 回合拦截逻辑】**：
1. **默认“需巩固”**：评估栏**必须**填入 `[需巩固]`，禁止填入分数或已掌握。
2. **第一轮提问**：AI 必须立即输出：“为了进行真实理解检验，我将发起连环追问。第一轮考点：<本单元基础概念 What/How>”，并出一道检验题。然后 **STOP**，等待用户回答。
3. **禁止行为**：禁止在第 1 回合直接输出任何预告模板、评分或跳转新单元。

### Step 2: 第一轮点评与第二轮追问（第 2 回合）

当用户提交第一轮解答后，AI 进入本回合：
1. AI 必须对用户的回答进行专业点评，纠正概念偏差，提供精炼的面试口径。
2. **第二轮追问（深度拷打）**：AI **必须**进行连环追问，发起第二轮考核。提问必须触及底层原理（如数学公式推导、源码实现类与方法定位、或具体业务决策 Why/Where），并 **STOP**，等待用户回答。

### Step 3: 终期评分与推进征求（第 3 回合）

当用户提交第二轮解答后，AI 进入本回合：
1. AI 对第二轮回答进行点评，打磨面试口吻。
2. **掌握度分级量化**：严格根据 `_GLOBAL_RULES.md` 规则 13 第 2 条的评分标准，给出量化评分（如 `3.5分` / `4.5分`）。
3. **输出跳转预告（仅限评分 >= 3.0分）**：若评分及格，输出 `Step 3-A` 模板，征求用户同意推进；若低于 3.0分，输出 `Step 3-C` 拒绝推进并重新开始巩固。然后 **STOP**。

### Step 4: 用户确认推进与状态落盘（第 4 回合）

当用户发送确认推进的指令后，AI 进入本回合（备份 → 写入 → 校验 → 失败回滚，按 `_GLOBAL_RULES.md` 规则 14）：
1. **写入前备份**：将 `docx/StudyState.json` 与活跃天的 `docx/StudyMemory/Day_N.md` 复制到 `docx/hooks/backup/`（同文件名加 `.bak` 后缀）。
2. **状态自动保存（WAL）**：
   - 修改 `docx/StudyState.json` 中当前单元状态为 `"completed"`，并将 rating 设为实际得分。
   - Edit 活跃天的 `docx/StudyMemory/Day_N.md`，将对应单元从 `[ ]` 改为 `[x]`，并在 `掌握度评分` 栏更新实际分值。
3. **强制执行校验 Hook**：
   - **必须**在终端运行校验命令：`python docx/hooks/validate_study.py`。
   - 若校验返回非零，**必须**用第 1 步的备份恢复上述所有文件更改，将具体的校验错误报告给用户并 **STOP**，终止本次跳转。
4. **加载新单元**：通过校验后，常规推进或超前学习至下一单元，输出新单元开场模板（Step 5）开始讲解。


### Step 3-C: 未掌握 → 拒绝推进（第 2 回合答题失败时输出）

<!-- template:reject_advance -->
```
我判断你还没掌握「<单元名>」的核心：
- 卡点：<具体卡点>
- 表现：<用户哪里没答上来 / 复述哪里有偏差>

我建议先巩固再推进。如果坚持跳过，说 [强制下一内容]，但会在 StudyMemory 标记为「未掌握-跳过」。
```

**禁止**：用户说 `[下一内容]` 时，即使 AI 判断未掌握也直接推进。**必须**先输出此模板。

### Step 3-D: `[强制下一内容]` 分支（跳过连环追问）

当用户在 Step 1-3 任意阶段明确说 `[强制下一内容]` 时：

1. **跳过连环追问**，不再等待两轮答题。
2. **标记薄弱**：掌握度按 `[低于3.0分 (薄弱)]` 处理：
   - `docx/StudyState.json` 中当前单元状态设为 `"completed"`，rating 记 `2.0`（或 AI 据实给出的 < 3.0 分值）。
   - Edit 活跃天 `docx/StudyMemory/Day_N.md`：对应单元改为 `[x]` 并在行尾追加备注 `（未掌握-跳过）`，「掌握度评分」栏记实际分值。
3. **仍须走 Step 4 完整落盘流程**（备份 → 写入 → 校验 → 失败回滚），校验通过后方可进入 Step 5。
4. 该单元自动加入当日 StudyMemory「明日优先项」，并在 `[开始今日复盘]` 时作为重点拷问题材。

### Step 5: 进入下一单元（或超前学习单元）

**常规推进**：
如果有未学完单元，输出新单元开场模板。

**「超前学习」分支**：
如果用户说 `[超前学习]`，或者今日单元已全部学完，且用户同意加载明日内容：
1. 从 `Study.md` 的 `Day N+1` 大纲中提取第一个学习单元。
2. 将该单元信息复制/拽入 `docx/StudyState.json` 的 `days[current_day].units` 列表中，标为 `"in_progress"`，并设置额外的标记 `"ahead": true`。
3. 在 `docx/StudyMemory/Day_N.md` 中的「今日导学单元」末尾追加一行：`- [ ] 单元 X（超前）：<标题>`。
4. 输出新单元开场模板开始讲解。

输出新单元开场模板格式（**必须**与 `SOP_开始今日学习.md` Step 4 统一格式，含开头和末尾 `---`，含所有 `**` 加粗）：

```
---【单元 N 开始】---
单元：<新单元名>
目标：<1 句话>
预计时长：<X>min
关联面试考点：<1-2 个>

我将带读以下材料：
- 项目文档：<路径>
- 业内论文/文章：<引用>

第一段：<开始讲解>
---
```

**模板硬约束**：
- 开头 `---【单元 N 开始】---` 必输
- 末尾 `---` 必输（单元开场块闭合）
- N = 当前单元序号（从 1 开始）
- 末尾 `---` 之后才进入讲解，缺 `---` = 违规
